2.9

Code 
sin(pi)
[1] 1.224606e-16

En R, cuando calculamos sin(pi) debería dar exactamente 0, porque matemáticamente sin(pi)=0 Sin embargo, lo que aparece en la consola es un número muy pequeño como 1.224606e-16. Esa notación significa 1.224606×10 a la menos 16, que en decimal es 0.0000000000000001224606. Este valor no es realmente distinto de cero, sino un error de redondeo que ocurre porque la computadora no puede representar pi con precisión infinita, solo con un número limitado de decimales. Al usar esa aproximación, el resultado no es exactamente cero, sino un número diminuto que se considera prácticamente cero.

3.5

Code 
e <- a+b

En este caso se crea una variable la cual es “e” la cual se crea sumando a= 1 y b=2 ese número queda guardado dentro de la variable “e”. Es decir, queda guardada en R.

3.6

Code 
(e <- a+b)
[1] 3

Hace exactamente la misma asignación guarda la suma de a+b en la variable e. La diferencia es que, al ponerlo entre paréntesis, R no solo realiza la asignación, sino que también muestra inmediatamente el resultado en pantalla.

3.7

Code 
d
[1] 3

Nos muestra el valor de “d” el cual es 3

3.8

Code 
f
[1] 1

El codigo f <<- 1L crea una variable llamada f y le asigna el valor 1 en tipo entero. La diferencia entre escribir 1 y 1L es que 1 normalmente se guarda como un número de tipo numerico ya sea decimal, mientras que 1L se guarda explícitamente como un entero. Además, el operador <<- no es igual a <- mientras <- asigna la variable en el entorno actual, <<- busca en entornos superiores y asigna allí, lo que significa que puede modificar variables globales o externas.

9 VECTORES X1-X2

Code

Code

[1]  7  9 11 10 12

Cuando en R sumas dos vectores que no tienen la misma cantidad de elementos, el programa repite los valores del vector más corto hasta que tenga la misma longitud que el más largo x1 tiene 5 números y x2 tiene 3. R repite x2 como 6,7,8,6,7 para que tenga 5 elementos, y luego suma cada posición: 1+6=7, 2+7=9, 3+8=11, 4+6=10, 5+7=12.
